HTTP & HTTPS Protocols:

Hypertext Transfer Protocol Secure (HTTPS) is the encrypted version of Hypertext Transfer Protocol (HTTP). HTTP (Hypertext Transfer Protocol) is an Application layer protocol that allows web-based applications to communicate. Application layer is in the OSI model with contain seven layers. These layers are the following: Physical Layer, Datalink Layer, Network Layer, Transport Layer, Session Layer, Presentation Layer, and Application Layer. Or as I remember them as “Please do not throw sausage pizza away”. Application layer use network application to exchange data on a network. For example, web browser like Chrome and Firefox, which all used HTTP protocol.

HTTP is a TCP/IP protocol call and is the messenger of the web. It used to deliver contents such as image, video, and document. In other word, HTTP can deliver any type of data as long as the receiving and sending computers can both read it. And because of that, HTTP protocol became the most convenient way to quickie send reliably data on the web. If two computers want to exchange data, they must follow the HTTP protocol which is basically a request response cycle. Namely the client is the computer that make the request and the server is the one responds to the request.

A HTTP request message contain the Start line (which include the Method, URI, and HTTP Version) and the Header (which contain the Host, Accept, and Accept Language). The Method is the command that the client give to the serve, instructing the server what to do. The URI show where we are getting our resource from. Basically, the location of the information from the server. The HTTP Version data is what version the client is using. The Host is the address of the server. Accept language tell you what language the information is and Accept tell you what type of file the data is in.

A HTTP respond message have a Start line (Which contain the HTTP version and the status code), the Header (which contain the Host, Accept, and Accept Language), and a respond body (that contain the file requested). The status code tell the client is the request is successful or not, which depend on the first digit of the status code. Notice that the Method and URL is not in the Start line because those are specifically for the request HTTP request message.
